# 채용 공고 추천 프롬프트 명세

## 목적

수집된 채용 공고를 사용자 관점에서 점수화하고 추천 이유를 생성한다.
현재 BE는 IT/소프트웨어 개발 직군 취준생 관점의 추천 프롬프트를 사용한다.

## 현재 BE 출처

| 기능 | 현재 파일 |
|------|-----------|
| 채용 공고 추천 평가 | `BE/src/recruit/application/recruit-job-posting-collect.service.ts` |

## 추천 평가 프롬프트

### User Prompt

````md
아래 채용 공고를 IT/소프트웨어 개발 직군 취준생 관점에서 평가해줘.
평가 기준: ① 개발 직무 관련성 ② 성장/학습 기회 ③ 경험 쌓기 적합도
각 공고에 0~100점을 부여하고, 추천 이유(한 문장)와 핵심 포인트(2~3가지) 작성.

반드시 JSON만 출력:
{"items":[{"id":"공고id","score":85,"reason":"추천 이유","match_points":["포인트1","포인트2"]}]}

공고 목록:
{JSON.stringify(rows)}
````

### 입력 데이터

```json
[
  {
    "id": "공고 ID",
    "title": "공고 제목",
    "company": "회사명",
    "jobs": "직무",
    "detail": "공고 상세 내용. 최대 1200자"
  }
]
```

### 응답 형식

```json
{
  "items": [
    {
      "id": "공고id",
      "score": 85,
      "reason": "추천 이유",
      "match_points": ["포인트1", "포인트2"]
    }
  ]
}
```

## BE_BROWSE 이전 시 고려사항

- 현재 프롬프트는 IT/소프트웨어 개발 직군 기준이므로, 장기적으로 사용자 프로필 기반 동적 평가 기준으로 확장한다.
- 추천 기준은 `직무 적합성`, `성장 가능성`, `경험 축적 가능성`, `사용자 기술 스택 적합성`으로 분리한다.
- JSON 응답은 파서에서 `id`, `score`, `reason`, `match_points`를 검증한다.
- 추천은 큐 작업으로 실행하고, 공고별 추천 점수와 이유를 저장한다.
