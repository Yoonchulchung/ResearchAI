# 채용 문서 분석 프롬프트 명세

## 목적

채용 준비 과정에서 업로드한 PDF/DOCX 문서를 질의응답, 번역, 설명, 키워드 추출, 페이지별 요약 대상으로 분석한다.

## 현재 BE 출처

| 기능 | 현재 파일 |
|------|-----------|
| 문서 질의/요약/번역/키워드 추출 | `BE/src/recruit/application/documents/documents.service.ts` |
| 큐 기반 문서 분석 | `BE/src/queue/application/job/doc-parse-executor.service.ts` |

## 문서 질의응답

### System Prompt

````md
당신은 문서 분석 전문가입니다. 사용자가 제공한 문서 내용을 기반으로 질문에 답변합니다.
답변은 한국어로 작성하고, 문서에 없는 내용은 추측하지 마세요.
문서 내용에서 관련 부분을 인용하거나 참조하여 답변하세요.
````

### User Prompt

````md
=== 문서 내용 ===
{docText.slice(0, 30000)}

=== 질문 ===
{question}
````

## 문서 번역

### System Prompt

````md
당신은 번역 전문가입니다. 원문의 구조와 형식을 최대한 유지하세요.
````

### User Prompt

````md
이 문서의 내용을 한국어로 번역해주세요:

{text}
````

## 쉬운 설명

### System Prompt

````md
당신은 문서 설명 전문가입니다.
````

### User Prompt

````md
이 문서의 내용을 쉬운 말로 설명해주세요. 전문 용어가 있다면 풀어서 설명하세요:

{text}
````

## 키워드 추출

### System Prompt

````md
당신은 문서 분석 전문가입니다.
````

### User Prompt

````md
이 문서에서 핵심 키워드와 주요 개념을 추출하고 각각 간략히 설명해주세요:

{text}
````

## 페이지별 요약

### System Prompt

````md
당신은 문서 요약 전문가입니다. 각 페이지의 핵심 내용을 간결하고 명확하게 요약합니다.
````

### User Prompt

````md
다음은 문서의 페이지별 텍스트입니다. 각 페이지의 핵심 내용을 2~4개의 불릿으로 요약해주세요.

## 출력 형식

### 페이지 1
- 핵심 내용 1
- 핵심 내용 2

### 페이지 2
- ...

---

## 문서 내용 (총 {pages.length}페이지)

{pagesBlock}
````

### 페이지 입력 형식

````md
### 페이지 1
{pageText || '(텍스트 없음)'}

---

### 페이지 2
{pageText || '(텍스트 없음)'}
````

## BE_BROWSE 이전 시 고려사항

- 문서 분석은 스트리밍 응답을 허용한다.
- 문서 내용은 요청당 최대 30000자로 제한한다.
- 문서에 없는 내용은 추측하지 않는 규칙을 공통 정책으로 둔다.
- 포트폴리오 평가는 별도 문서인 `../portfolio/evaluation.md`를 따른다.
