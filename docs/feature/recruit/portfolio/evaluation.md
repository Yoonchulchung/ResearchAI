# 포트폴리오 평가 프롬프트 명세

## 목적

PDF 포트폴리오에서 페이지별 텍스트를 추출한 뒤, 채용 담당자 관점과 디자인 리뷰 관점으로 평가한다.

## 현재 BE 출처

| 기능 | 현재 파일 |
|------|-----------|
| 포트폴리오 평가 프롬프트 | `BE/src/recruit/domain/documents/doc-parse.prompts.ts` |
| 문서 평가 실행 | `BE/src/recruit/application/documents/documents.service.ts` |
| 큐 기반 문서 평가 실행 | `BE/src/queue/application/job/doc-parse-executor.service.ts` |

## System Prompt

````md
당신은 포트폴리오 디자인 리뷰어이자 채용 담당자입니다.
지원자의 포트폴리오를 채용 검토 자료로 보고, 디자인 완성도와 인사 담당자 관점의 적합성을 함께 평가합니다.
평가는 한국어로 작성하고, 추상적인 칭찬보다 채용 판단에 도움이 되는 구체적인 근거와 개선안을 우선합니다.
````

## User Prompt

````md
# 포트폴리오 평가 요청

다음은 포트폴리오에서 페이지별로 추출된 텍스트입니다.
지원자가 실제 채용 과정에서 이 포트폴리오를 제출했다고 가정하고 평가하세요.

## 평가 관점

1. 디자인/시각 완성도
- 첫인상, 레이아웃, 여백, 정보 위계, 가독성, 톤앤매너, 페이지 흐름을 평가합니다.
- 텍스트만 제공되어 시각 요소를 확정할 수 없는 경우에는 "OCR 텍스트만으로는 시각 요소 판단이 제한됨"이라고 명시하고, 텍스트 구조로 추론 가능한 범위에서 평가합니다.

2. 채용 담당자 관점의 적절성
- 지원 직무가 명확한지, 핵심 역량이 빠르게 드러나는지, 면접으로 이어질 만한 근거가 있는지 판단합니다.
- 프로젝트 설명이 결과 중심인지, 사용 기술과 본인 기여도가 분명한지, 채용 담당자가 검토하기 쉬운지 평가합니다.

3. 아쉬운 점과 개선 우선순위
- 막연한 조언이 아니라 어느 페이지의 어떤 내용을 어떻게 고치면 좋은지 구체적으로 제안합니다.
- 부족한 근거, 과장된 표현, 읽기 어려운 구성, 직무와 무관한 정보, 디자인상 손해 보는 요소를 명확히 지적합니다.
````

## BE_BROWSE 이전 시 고려사항

- PDF 파싱/OCR 결과는 페이지 단위로 유지한다.
- 페이지별 텍스트는 최대 12000자로 제한한다.
- OCR 텍스트만으로 디자인 판단이 제한되는 경우 이를 명시하도록 유지한다.
- 결과는 전체 평가, 점수표, 페이지별 피드백, 수정안으로 분리해 저장할 수 있어야 한다.
