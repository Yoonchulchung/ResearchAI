# 뉴스/콘텐츠 기능 명세

## 목적

사용자가 뉴스, 논문, 기술 블로그, AI 리더보드 정보를 확인하고 요약/트렌드 결과를 탐색할 수 있게 한다.

## 주요 화면

| 경로 | 설명 |
|------|------|
| `/news` | 뉴스 홈 |
| `/news/feed` | 뉴스 피드 |
| `/news/papers` | 논문 목록 |
| `/news/papers/[paperId]` | 논문 상세 |
| `/news/tech-blogs` | 기술 블로그 |
| `/news/leaderboard` | AI 리더보드 |
| `/news/leaderboard/[modelId]` | 모델 상세 |

## 뉴스 피드

### 기능 요구

- 사용자는 카테고리별 뉴스를 조회할 수 있다.
- 시스템은 뉴스 기사 요약을 제공한다.
- 주요 이슈나 분쟁 지역 정보는 대시보드/지도와 연결될 수 있다.

## 논문

### 기능 요구

- 사용자는 논문 목록을 조회한다.
- 논문 상세에서 초록, 요약, 트렌드 정보, 관련 메타데이터를 확인한다.
- 논문 요약/트렌드 작업은 큐로 실행될 수 있다.

## 기술 블로그

### 기능 요구

- 사용자는 기술 블로그 포스트 목록과 요약/트렌드를 확인한다.
- 트렌드 요약은 별도 큐 작업으로 생성될 수 있다.

## AI 리더보드

### 기능 요구

- 사용자는 AI 모델 리더보드 목록을 확인한다.
- 특정 모델 상세에서 성능/메타 정보를 확인한다.

## 주요 API/모듈

| 영역 | 설명 |
|------|------|
| `news.controller` | 뉴스 피드/요약 |
| `papers.controller` | 논문 목록/상세 |
| `tech-blog.controller` | 기술 블로그 |
| `ai-leaderboard.controller` | AI 리더보드 |
| `news-summary.service` | 뉴스 요약 |

## 예외/빈 상태

- 외부 뉴스 수집 실패 시 마지막 성공 데이터를 유지할 수 있다.
- 요약이 없으면 원문/메타데이터만 표시하고 요약 생성 상태를 표시한다.
- 모델 상세가 없으면 리더보드 목록으로 이동할 수 있어야 한다.
