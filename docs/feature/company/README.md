# 기업 기능 명세

## 목적

사용자가 기업 목록과 상세 정보를 관리하고, 웹 검색/크롤링/AI 분석을 통해 기업 정보와 채용 관련 인사이트를 보강할 수 있게 한다.

## 주요 화면

| 경로 | 설명 |
|------|------|
| `/companies` | 기업 목록 |
| `/companies/[id]` | 기업 상세 |
| `/companies/analysis` | 기업 분석 실행/조회 |
| `/company-analysis` | 기존 기업 분석 화면 |

## 기업 목록

### 기능 요구

- 사용자는 기업 목록을 조회한다.
- 기업명, 산업, 관심도, 분석 상태, 최신 보강 시간을 확인할 수 있다.
- 기업을 선택하면 상세 화면으로 이동한다.

## 기업 상세

### 기능 요구

- 기업 기본 정보, 재무 정보, 채용/HR 정보, 뉴스/자료를 표시한다.
- 신뢰도나 출처 정보가 있는 데이터는 사용자에게 근거를 제공한다.
- 기업 관련 채용 공고 또는 자기소개서 작성과 연결될 수 있다.

## 기업 정보 보강

### 기능 요구

- 시스템은 외부 출처에서 기업 정보를 수집한다.
- 출처에는 DART, 사람인, 잡코리아, 잡플래닛, 자소설닷컴, 나무위키, 부동산/입지 정보 등이 포함될 수 있다.
- 보강 작업은 큐로 실행되며 상태를 추적한다.
- 누락된 기업 정보는 별도 refresh 작업으로 보강할 수 있다.

## 기업 분석

### 기능 요구

- 사용자는 기업 분석을 요청할 수 있다.
- 시스템은 기업 정보, 뉴스, 재무, 채용 페이지, 외부 검색 결과를 조합해 분석한다.
- 분석 결과는 점수/요약/근거/리스크/추천 액션을 포함할 수 있다.
- 분석 결과는 기업 상세에서 재조회 가능해야 한다.

## 주요 API/모듈

| 영역 | 설명 |
|------|------|
| `companies.ts` | FE 기업 API wrapper |
| `company.controller` | 기업 CRUD/조회 |
| `company-analysis.controller` | 기업 분석 |
| `company-enrich-queue.service` | 기업 보강 큐 |
| `career-page-url.service` | 채용 페이지 탐색 |

## 예외/빈 상태

- 기업 정보가 부족하면 “정보 부족” 상태와 보강 실행 버튼을 표시한다.
- 외부 출처 실패는 출처별 실패로 기록한다.
- 분석 결과가 오래된 경우 재분석을 유도한다.
